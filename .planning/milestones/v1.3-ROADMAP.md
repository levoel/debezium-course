## Overview

v1.3 applied liquid glass (glassmorphism) design to the Debezium course website and improved navigation with descriptive module names and accordion-based homepage. This milestone transformed the visual experience while maintaining WCAG accessibility standards and mobile performance.

## Accomplishments

### Phase 22: Foundation (Glass Design System + Module Naming)
- CSS custom properties centralize all glass parameters (blur, opacity, shadows)
- Three glass variants: standard panel, elevated panel, sidebar
- Vibrant gradient background (purple/blue/pink/cyan orbs)
- Descriptive module names single source of truth (moduleNames.ts)
- Accessibility support via prefers-reduced-transparency and prefers-reduced-motion

### Phase 23: Homepage Redesign (Accordion + Module Cards)
- ModuleAccordion component with glass-styled cards
- All modules collapsed by default for cleaner initial view
- Per-module progress indicators (calculated from lesson checkmarks)
- Hover lift animation (-2px translateY) for interactive cards

### Phase 24: Content Components (Tables + Callouts)
- Light glass effect for tables (8px blur, low opacity)
- Clear 1px solid cell borders for data tracking
- Solid code blocks (no glass) to preserve syntax highlighting
- Glass callouts with type-specific colored tints (blue/green/yellow/red)

### Phase 25: Polish (Mobile + Accessibility + Verification)
- Safari CSS variable bug fixed with hardcoded webkit values
- Mobile blur reduction (8px vs 10px-16px desktop)
- Automated accessibility testing with @axe-core/playwright
- WCAG 2.1 AA compliance verified (6 tests)
- Color contrast fixes for navigation and topic tags

### Quick Tasks (User Acceptance Testing)
- Fixed module order in sidebar with explicit sort
- Shortened module names for one-line display
- Centered course content area
- Apple liquid glass styling refinements (muted colors, glass code blocks)

## Key Decisions

1. **CSS custom properties** centralize all glass parameters for consistency
2. **Three glass variants** (standard, elevated, sidebar) for different UI contexts
3. **Mobile blur reduction** (10px â†’ 8px) for performance on mobile devices
4. **Accessibility-first** - glass effects are enhancement, not requirement
5. **Module names** stored in single source of truth (moduleNames.ts)
6. **Light glass for tables** (8px blur) to avoid overwhelming data content
7. **Solid code blocks** (no glass) to preserve syntax highlighting readability
8. **Safari workaround** - hardcoded webkit values due to Safari 18 CSS variable bug
9. **Exclude Shiki** code blocks from accessibility tests (third-party theme limitation)
